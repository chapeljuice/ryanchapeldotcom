New site! Now with *REAL* content!

May 12, 2014

For the longest time ryanchapel.com was basically just my name and some contact info. I've finally taken the time to build something that I think is nice, and also put some content on here that I *should* be updating regularly!

All that being said, let's talk about the site a bit:

RWD - Responsive Web Design

Everybody's favorite. I guess my last design was technically responsive, althogh nothing really moved around that much or behaved differently at different browser widths. This time I've taken the time to develop the site using the mobile first approach—that is, start with designing / developing the mobile experience, and then get wider and wider until it looks good at my maximum width (which I set to 140rem, but we'll get to that in a second).

I tried to get away from using 'px' units for everything this time, and so I'm using mostly 'rem' units (for widths at least). Rem units are great because they're based off of your root size. For example, most browsers default their font-size to 16px. I set my font-size on my html element to be 62.5%, which (if you do the math) works out to be 10px. This keeps everything simple and straightforward, and so now whenever I need to specify something that's around 14px I'd just use 1.4rem, which is 14 / 10 (the base font-size). The only drawback (if you call it that) is that this is not supported in IE8 or below. I've decided not to support those browsers, so it's not a big deal here. If you were to support them, all you'd have to do is declare a font-size in a different unit right above the rem declaration, such as px or em.

CSS Transitions / Animations

If you look hard enough you'll see some subtle css transitions and animations that I think add to the user's experience, but not detract from it. In large browsers it's mostly subtle hover effects, however in smaller browsers when you scroll down the logo and other elements in the header shrink and scoot up a bit so that the user has more area to view the content.

Content

As I stated before I'm now going to be using this website not really as a 'blog', but as a way to teach users some of my experiences and how to create cool things I've thought of or come across. My hope is that I can help someone create a better user experience for their users on their sites, and also teach a few things in the process (heck, and maybe even learn a thing or two myself!)
